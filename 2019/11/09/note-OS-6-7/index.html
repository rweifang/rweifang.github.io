發表於 2019-11-09 分類於 Operating System Disqus：

Monitors

先前介紹了mutex和semaphore，但是錯誤的使用它們會導致嚴重的錯誤。接下來我們介紹更高階的同步工具monitor。monitor將mutex或semaphore的功能封裝成一個物件，只能透過monitor提供的函數對其操作。

Condition variable

首先介紹monitor中會用到的data type, condition，condition可以使用的function為wait()以及signal()。
使用wait()的process會被暫停，而呼叫signal()會使被暫停的process繼續，如果沒有被暫停的process則沒有效果。

Condition variable中的signal()喚醒的對象可以使用first come, first serves (FCFS)的策略，或是使用wait(c)對暫停的process加入權重。
