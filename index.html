Note-OS-8.5 Deadlock Prevention

發表於 2019-11-10 分類於 Operating System Disqus：

Deadlock Prevention

這裡提出一些方法，使deadlock的四種必要條件之一不會發生，進而避免deadlock的發生。

Mutual Exclusion

對於一個不可分享的資源來說，這個條件不可能避免。

Hold and Wait

避免這個條件的第一種方法是，在執行之前，一次獲得所有需要的資源。這個方法會使得資源的使用率很低。
第二個方法是，必須先釋放所有資源，才可以獲得新的資源。這個方法有機會導致starvation。

大致上來說，處理deadlock的方法有以下幾種：

  • 忽視問題，假裝deadlock沒有發生在系統中。

Note-OS-8.3 Deadlock Characterization

發表於 2019-11-10 分類於 Operating System Disqus：

Deadlock Characterization

Necessary Conditions

只有在下列四個條件都成立時，deadlock才會發生。

  1. Mutual exclusion 至少有一個不可分享的資源，也就是一次只有一個thread可以使用這個資源。
  2. Hold and wait 當某一個thread持有一個資源，並且等待另一個資源的釋放。
  3. No preemption 資源只能被自願的釋放，不能被其它thread搶走。
  4. Circular wait 當一組thread $T={T0,T_1,…,T_n}$，$T_0$等待$T_1$釋放資源，$T_1$等待$T_2$，…，而$T{n-1}$等待$T_n$，$T_n$等待$T_0$釋放資源。

Note-OS-6.4 Hardware Support for Synchronization

發表於 2019-11-09 分類於 Operating System Disqus：

Hardware Support for Synchronization

Memory Barrier

電腦架構中的memory model大致上可以分為下列兩種

  1. Strongly ordered, 在某一個處理器中做修改記憶體的值，會立刻更新到其它處理器。
  2. Weakly ordered, 在某一個處理器中做修改記憶體的值，不會立刻更新到其它處理器。

Memory model隨處理器而異，有可能是Strogly ordered，也有可能是Weakly ordered。透過memory barrier或memory fences的instruction可以強制更新。
